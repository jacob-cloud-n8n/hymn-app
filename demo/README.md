## 🎬 展示 SOP（給 Home OC 自己用）

### 1a — 檔案分類整理（已驗證 ✅）

**目標**：在桌面上建立一個混亂資料夾，當客戶面整理。

**流程**：
```
1. 建立桌面資料夾 → `mkdir -p ~/Desktop/AI_Demo_整理前/` + 放入亂檔
2. 執行 `open ~/Desktop/AI_Demo_整理前/` → 讓客戶看 Finder 混亂狀態
3. ⏸️ 停，等客戶說「可以整理了」
4. 分類：建立 01_~05_ 資料夾 + mv 歸位
5. ⏸️ 停，等客戶看 Finder
6. 檔名清理：移除「真ㄉ是最後版」「final-final」等前綴
7. 展示完成
```

**提醒**：每步做完要停，不要一口氣做完。讓客戶在 Finder 親眼看到變化。

### 3a — 網頁抓取分析（已驗證 ✅）

**目標**：客戶給一個 URL，AI 抓取內容並產出 HTML 分析報告。

**流程**：
```
1. 客戶提供網址
2. 執行 `webfetch` 抓取網頁內容（約 3–5 秒）
3. 分析頁面結構、產品/服務、重點資訊
4. 產出 HTML 分析報告（含 AI 觀點總結）
5. 執行 `open ~/Desktop/AI_Demo_{主題}.html` → 瀏覽器展示
```

**限制**：
| 類別 | 可抓取 | 不可抓取 |
|------|--------|---------|
| 靜態網頁 | ✅ 完整內容 | — |
| 需登入網站 | — | ❌ 無法通過認證 |
| JS 渲染頁面 | — | ❌ 僅能抓取原始 HTML |

### 3b — 方案優缺對照表（已驗證 ✅）

**目標**：客戶給一個主題，現場搜尋知識庫 + 產出 HTML 比較表。

**流程**：
```
1. 客戶說主題（如「LINE」「自架 vs SaaS」）
2. 先從 KB-Vault grep 搜尋相關資料
3. 若客戶沒指定比較對象 → 主動提 2–3 組常見組合讓選
4. 產出 HTML 比較表（~10 項維度 + 結論區塊）
5. 執行 `open ~/Desktop/AI_Demo_方案比較.html` → 瀏覽器展示
```

**格式選擇**：
| 格式 | 指令 | 客戶體驗 |
|------|------|---------|
| HTML ✅ | `寫入 .html → open` | 瀏覽器開啟，排版最好 |
| CSV | `寫入 .csv → open` | Excel/Numbers 開啟 |
| MD | `寫入 .md` | 需編輯器，較不友善 |

**提醒**：產出後停在瀏覽器畫面，讓客戶自己滾動閱讀。
